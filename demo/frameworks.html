Framework Integration — Use DTF Anywhere

Design Token Forge is CSS-first and framework-agnostic. Our tokens are CSS custom properties, our components are pure CSS with data attributes. Your framework handles logic — our CSS handles every pixel.

React Vue Angular Svelte Solid Web Components Vanilla JS Zero Runtime

Core Principle

Design Token Forge separates visual styling (CSS custom properties + data-attribute selectors) from component logic (events, state, reactivity). Your framework owns the logic. Our CSS owns every visual property.

🎨

Tokens = CSS Vars

All design tokens are native CSS custom properties. They work everywhere CSS works — no build step, no runtime, no JS overhead.

📐

Data Attributes

Variants and sizes use data-variant and data-size — which map 1:1 to your component props.

🔌

Import CSS, Write Props

Import our token + component CSS files. Then write thin wrapper components that pass props as data attributes.

⚡

Zero Runtime Cost

No CSS-in-JS, no style injection, no theme provider. Pure CSS that the browser already knows how to optimize.

How Props Map to CSS

The key insight: your framework component props translate directly to HTML data-* attributes. Our CSS selectors do the rest.

Your PropHTML OutputDTF CSS SelectorEffect
variant="brand" data-variant="filled" data-role="brand" [data-variant="filled" data-role="brand"] Applies brand |brand | danger | warning | info | success | neutral|brand | danger | warning | info | success | neutral| info |brand | danger | warning | info | success | neutral|brand | danger | warning | info | success | neutralsurface colors
size="large" data-size="large" [data-size="large"] Sets height, padding, font-size, radius
disabled data-disabled + disabled [data-disabled] Applies disabled opacity + cursor
error data-error [data-error] Switches to error border + text colors
loading data-loading [data-loading] Shows shimmer / spinner animation
That's the whole pattern. Every DTF component uses the same convention: props become data attributes, our CSS selectors match them, and the browser renders everything. No JS runtime needed for styling.

Select your framework to see how to wrap DTF components. Each wrapper is typically 10-20 lines.

DTFButton.tsx

React + TypeScript
import { forwardRef } from 'react';

type Variant = 'brand' | 'secondary' | 'ghost' | 'danger' | 'outline';
type Size = 'micro' | 'tiny' | 'small' | 'base' | 'medium' | 'large' | 'big' | 'huge' | 'mega' | 'ultra';

interface DTFButtonProps extends React.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: Variant;
  size?: Size;
  loading?: boolean;
}

export const DTFButton = forwardRef<HTMLButtonElement, DTFButtonProps>(
  ({ variant = 'brand', size = 'base', loading, disabled, children, ...rest }, ref) => (
    <button
      ref={ref}
      className="btn"
      data-variant={variant}
      data-size={size}
      data-loading={loading || undefined}
      data-disabled={disabled || undefined}
      disabled={disabled}
      {...rest}
    >
      <span className="btn__label">{children}</span>
    </button>
  )
);

DTFInput.tsx

React + TypeScript
interface DTFInputProps extends React.InputHTMLAttributes<HTMLInputElement> {
  variant?: 'outline' | 'filled' | 'underline';
  size?: Size;
  label?: string;
  helper?: string;
  error?: boolean;
}

export const DTFInput = forwardRef<HTMLInputElement, DTFInputProps>(
  ({ variant = 'outline', size = 'base', label, helper, error, disabled, ...rest }, ref) => (
    <div
      className="input-group"
      data-variant={variant}
      data-size={size}
      data-error={error || undefined}
      data-disabled={disabled || undefined}
    >
      {label && <label className="input__label">{label}</label>}
      <div className="input-field">
        <input ref={ref} disabled={disabled} {...rest} />
      </div>
      {helper && <span className="input__helper">{helper}</span>}
    </div>
  )
);

DTFInput.vue

Vue 3 SFC
<script setup lang="ts">
defineProps<{
  variant?: 'outline' | 'filled' | 'underline'
  size?: string
  label?: string
  helper?: string
  error?: boolean
  disabled?: boolean
  modelValue?: string
}>()
defineEmits(['update:modelValue'])
</script>

<template>
  <div class="input-group"
    :data-variant="variant ?? 'outline'"
    :data-size="size ?? 'base'"
    :data-error="error || undefined"
    :data-disabled="disabled || undefined">
    <label v-if="label" class="input__label">{{ label }}</label>
    <div class="input-field">
      <input :value="modelValue" @input="$emit('update:modelValue', ($event.target as HTMLInputElement).value)" :disabled="disabled" />
    </div>
    <span v-if="helper" class="input__helper">{{ helper }}</span>
  </div>
</template>

DTFButton.tsx

SolidJS
import { splitProps } from 'solid-js';
import type { JSX } from 'solid-js';

interface Props extends JSX.ButtonHTMLAttributes<HTMLButtonElement> {
  variant?: string;
  size?: string;
  loading?: boolean;
}

export function DTFButton(props: Props) {
  const [local, rest] = splitProps(props, ['variant', 'size', 'loading', 'children']);

  return (
    <button
      class="btn"
      data-variant={local.variant ?? 'brand'}
      data-size={local.size ?? 'base'}
      data-loading={local.loading || undefined}
      {...rest}
    >
      <span class="btn__label">{local.children}</span>
    </button>
  );
}

dtf-button.js

Custom Element
class DtfButton extends HTMLElement {
  static observedAttributes = ['variant', 'size', 'disabled', 'loading'];

  connectedCallback() { this.render(); }
  attributeChangedCallback() { this.render(); }

  render() {
    const v = this.getAttribute('variant') || 'brand';
    const s = this.getAttribute('size') || 'base';
    const d = this.hasAttribute('disabled');

    this.innerHTML = `
      <button class="btn" data-variant="${v}" data-size="${s}"
        ${d ? 'data-disabled disabled' : ''}>
        <span class="btn__label">${this.textContent}</span>
      </button>
    `;
  }
}
customElements.define('dtf-button', DtfButton);
No Shadow DOM needed. Since DTF CSS is global (custom properties cascade everywhere), you can use Light DOM and the styles just work. If you use Shadow DOM, adopt the stylesheets via adoptedStyleSheets.

Customizing Tokens

Override any token at any scope — globally, per-section, or per-instance. No build step, no theme provider.

CSSOverride examples
/* Global override — affects all buttons */
:root {
  --btn-radius-base: var(--radius-full);  /* pill-shaped buttons */
}

/* Scoped override — only inside this container */
.my-sidebar {
  --btn-height-base: var(--spacing-32);  /* compact buttons */
  --btn-font-size-base: var(--font-size-11);
}

/* Instance override — inline on one element */
<button class="btn" data-variant="brand"
  style="--btn-filled-bg: #8B5CF6">
  Custom Purple
</button>

Import Strategies

Choose the approach that fits your build setup.

StrategyHowBest For
Full bundle @import '@dtf/components' Apps using most components
Per-component @import '@dtf/components/button' Tree-shaking conscious apps
Tokens only @import '@dtf/tokens' Custom components using DTF tokens
CDN <link href="cdn/tokens.css"> Prototypes, no-build setups
Copy CSS Copy .tokens.css + .css files Vendor-lock-free, full ownership
Tokens CSS is always required. Component CSS files reference global tokens (spacing, radius, colors). Always import the tokens package first.
