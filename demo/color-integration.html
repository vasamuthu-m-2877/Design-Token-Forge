Color System Integration

How to use the 4-tier DTF color system in CSS, React, Vue, and any framework. From raw primitives to contextual surfaces — every tier explained with working code.

Token Architecture Overview

The DTF color system is built in four tiers. Each tier references the one below it via CSS custom property aliases. You should consume the highest tier that matches your use case — never reach down to primitives directly.

T0 Primitives
--prim-monochromatic-500 --prim-danger-300 --prim-warning-600
↓ referenced by
T1 Semantic
--brand-content-default --danger-component-bg-default --warning-on-container
↓ consumed by
T2 Surfaces
--surface-bright-bg --surface-base-ct-default --surface-dim-cm-bg
↓ consumed by
T3 Status
T1 tokens re-mapped per primary · brand · success warning · danger · info

T0 Primitives — Raw Palette Values

176 raw color swatches across 7 palettes (monochromatic, danger, warning, success, info, desaturated, brand). These are the source-of-truth hex values. Avoid using T0 tokens directly in production code — use T1/T2 instead for theme support.

When to use T0 directly: Only for debugging, palette visualization, or building your own token mapping layer. If you reference T0 tokens in CSS, your UI will not adapt when switching between light/dark themes.

Naming Convention

Pattern
/* --prim-{palette}-{step} */

--prim-monochromatic-500   /* Primary palette, step 500 */
--prim-danger-300          /* Danger palette, step 300 */
--prim-warning-white       /* Warning palette, lightest */
--prim-desaturated-700     /* Neutral/desaturated, step 700 */

/* Steps: white, 25, 50, 75, 100, 150, 175, 200, 250, 300, 350,
           400, 450, 500, 550, 600, 700, 750, 800, 850, 900, black */
CSSUsing primitives (rare — debugging only)
/* Import the token layer */
@import '@design-token-forge/tokens/src/primitives.css';

/* Direct primitive use — does NOT auto-switch in dark mode */
.debug-swatch {
  background: var(--prim-monochromatic-500);
  color: var(--prim-monochromatic-white);
}

/* Palette visualization */
.palette-strip--primary [data-step="300"] {
  background: var(--prim-monochromatic-300);
}
.palette-strip--danger [data-step="500"] {
  background: var(--prim-danger-500);
}

  return (

        width: 48, height: 48, borderRadius: 6

    />
  );
}

T1 Semantic Tokens — Role-Based Colors

252 tokens across 5 roles (primary, danger, warning, success, info) × 2 modes (light/dark). These are the main tokens you should use for text, icons, buttons, containers, and interactive elements. They auto-switch between light and dark themes.

Anatomy of a T1 Token

Pattern
/* --{role}-{category}-{variant} */

/* CONTENT — text, icons, labels */
--brand-content-default     /* default text/icon color */
--brand-content-strong      /* emphasized / heading text */
--brand-content-subtle      /* secondary / hint text */
--brand-content-faint       /* disabled / placeholder */

/* COMPONENT — filled interactive elements */
--brand-component-bg-default   /* button/chip fill */
--brand-component-bg-hover     /* hover state */
--brand-component-bg-pressed   /* active/pressed state */
--brand-component-outline-default  /* outline variant border */
--brand-component-outline-hover    /* outline hover */
--brand-component-outline-pressed  /* outline pressed */
--brand-component-separator   /* divider inside component */

/* CONTAINER — background panels, cards, banners */
--brand-container-bg        /* subtle background fill */
--brand-container-hover     /* container hover */
--brand-container-pressed   /* container pressed */
--brand-container-outline   /* container border */
--brand-container-separator /* divider inside container */

/* ON — text on filled surfaces */
--brand-on-component   /* white text on filled button */
--brand-on-container   /* dark text on tinted container */

/* Replace "brand" with any role: danger, warning, success, info */

/** Role-agnostic status card — pass any T1 role */
export function StatusCard({ role, children }: {
  role: 'brand' | 'danger' | 'warning' | 'success' | 'info';
  children: React.ReactNode;
}) {
  // Map role to CSS custom properties via inline styles
  const style = {
    background: `var(--${role}-container-bg)`,
    border: `1px solid var(--${role}-container-outline)`,
    color: `var(--${role}-on-container)`,
    padding: '16px',
    borderRadius: '8px',
  };

  return <div style={style}>{children}</div>;
}

// <StatusCard role="danger">Something went wrong</StatusCard>
// <StatusCard role="success">Payment confirmed</StatusCard>
ReactDark mode toggle
/** Toggle dark mode — all T1/T2 tokens auto-switch */
export function ThemeToggle() {
  const [dark, setDark] = useState(false);

  const toggle = () => {
    const next = !dark;
    setDark(next);
    document.documentElement.setAttribute(
      'data-theme', next ? 'dark' : 'light'
    );
  };

  return <button onClick={toggle}>{dark ? 'Light' : 'Dark'} Mode</button>;
}

T2 Surface Tokens — Page & Layout Colors

9 surface contexts with 16 tokens each = 144 tokens (+ dark mode variants). Use these for page backgrounds, sidebars, cards, modals, and any layout region. They provide consistent bg, content (ct-*), and component (cm-*) colors within that surface.

Surface Contexts

ContextUse ForRelative Brightness

T3 Status Tokens — Contextual Role Remapping

T3 is a Figma-side concept where T1 role tokens are re-bound per status context (primary, brand, success, warning, danger, info). In CSS, you consume T3 by scoping T1 tokens to a data attribute, so the same component renders with different role colors depending on context.

Key insight: T3 does not introduce new token names — it remaps --brand-* to the appropriate role's values based on a data-status attribute. This means any component using --brand-* tokens automatically adopts the correct role colors when placed inside a [data-status] context.
CSST3 status context setup
/* T3 remaps --brand-* per status context.
   Components that use --brand-* auto-switch color. */

/* Define the remap rules: */
[data-status="danger"] {
  --brand-content-default: var(--danger-content-default);
  --brand-content-strong: var(--danger-content-strong);
  --brand-component-bg-default: var(--danger-component-bg-default);
  --brand-component-bg-hover: var(--danger-component-bg-hover);
  --brand-component-bg-pressed: var(--danger-component-bg-pressed);
  --brand-on-component: var(--danger-on-component);
  --brand-container-bg: var(--danger-container-bg);
  --brand-container-outline: var(--danger-container-outline);
  --brand-on-container: var(--danger-on-container);
  /* … and so on for all --brand-* tokens */
}

[data-status="success"] {
  --brand-content-default: var(--success-content-default);
  --brand-component-bg-default: var(--success-component-bg-default);
  --brand-on-component: var(--success-on-component);
  /* … */
}

[data-status="warning"] {
  --brand-content-default: var(--warning-content-default);
  --brand-component-bg-default: var(--warning-component-bg-default);
  --brand-on-component: var(--warning-on-component);
  /* … */
}

/* Now use it: */
<div data-status="danger">
  <!-- All children using --brand-* render in danger colors -->
  <button class="btn" data-variant="brand">Delete Account</button>
</div>

<div data-status="success">
  <!-- Same button, but green -->
  <button class="btn" data-variant="brand">Confirmed</button>
</div>
React / TSXStatusContext component
import '@design-token-forge/tokens';
import './status-remap.css'; // the [data-status] rules above

type Status = 'brand' | 'danger' | 'warning' | 'success' | 'info';

/** Wrap any content to remap primary tokens to a status role */

  status: Status;
  children: React.ReactNode;
}) {
  return (
    <div data-status={status}>
      {children}

  );
}

Which Tier Should I Use?

Building…Use TierExample Token
Page background, sidebar, headerT2 Surface--surface-base-bg
Body text, headings on a pageT2 Surface--surface-base-ct-default
Branded button, link, active tabT1 Semantic--brand-component-bg-default
Error message textT1 Semantic--danger-content-default
Alert/banner backgroundT1 Semantic--warning-container-bg
Neutral input, table borderT2 Surface--surface-bright-cm-outline
Card on a pageT2 Surface--surface-container-bg
Dropdown/popoverT2 Surface--surface-float-bg
Same component, different status per contextT3 Status[data-status="danger"]
Palette visualizer, debug toolsT0 Primitive--prim-monochromatic-500
Rule of thumb: Use T2 for layout structure (backgrounds, text, neutral elements), T1 for branded/semantic meaning (actions, alerts, status), and T3 for contextual role switching. Only reach for T0 in tools that visualize the palette itself.
