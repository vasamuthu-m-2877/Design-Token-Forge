Onboard — brand → roles → themes

Closes WT findings T1, T2, T3 — one brand color was never enough

What changed: v0.1's onboard treated the brand color as a single input. Real projects like writer-handhelds ship 8 roles, 2 themes (light + dark with independent step indices), and 8 surface levels. v0.2 reframes onboard as 3 sub-steps. The surface ladder is now a first-class step, not an afterthought.

Annotations

  • Brand color step now anchors via "Exact" mode by default — closes the user-picks-blue-sees-different-blue regression we hit in editor v2.
  • Role mapping step is mandatory. 6 required roles + 2 optional. Each gets its own 11-step ladder via the palette engine.
  • Surface ladder is a first-class step. 8 levels, with semantic names (bright/base/dim/deep + container/over-container/float/inverse).
  • Theme pair lets light + dark pick different step indices per role — required for WCAG across both themes.
  • Onboard "Finish" leads into the v0.2 kit picker (platform × shape), not the v0.1 6-archetype grid.
