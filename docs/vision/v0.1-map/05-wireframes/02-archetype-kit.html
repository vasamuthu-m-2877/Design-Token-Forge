The remix contract

The kit PROMISES
  • Every L2 uses only DTF L1 + tokens (zero hardcoded values)
  • Every L2 has all states (default · hover · loading · empty · error)
  • Every L2 is fully responsive (4 viewports tested)
  • Every L2 has a code equivalent (React / Vue / Svelte)
  • Every L2 is RTL-safe and a11y-compliant
The kit DOES NOT promise
  • That you'll use every L2 (use what fits, ignore the rest)
  • That you can't replace any of them (detach + redesign freely)
  • That your product will look like the examples
  • Coverage for non-dashboard use cases (those have their own kits)
  • That every pattern you need exists today (compose your own L2s)

Recipes are docs, not components. Cheap to add (10 min). Promote to L2 only when used 3+ times across products. 1
  • This is the key architectural choice. Most "design systems" fail by componentizing every pattern. Recipes preserve flexibility while still being on-system.
