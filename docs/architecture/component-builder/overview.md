# Component Builder — Overview

A DTF component is built as **two parallel deliverables** that must agree:

1. **CSS layer** — `packages/components/src/{name}/`
   - `{name}.tokens.css` — public custom properties (designer override surface)
   - `{name}.css` — structural rules + internal `--_{xx}-*` switches
2. **Figma layer** — generated by the plugin from a **BLUEPRINT** object in
   `packages/figma-plugin/code.js` into a Component Set per family per master.

Both layers share the same vocabulary: variants, axes, tokens, slot rules. The
docs in this folder define that vocabulary.

## The 7 axes (every component must consider them)

| Axis | What it controls |
|---|---|
| **Shape** | Border radius, including the rounded boolean |
| **Dimension** | Height, width, padding, gap |
| **Surface** | Background, border, divider color |
| **Typography** | Font family, weight, size, line-height |
| **Slots** | Icon / text / chevron / divider — what content goes where |
| **Motion** | Hover/press transitions, focus animation |
| **A11y** | Focus ring, min tap target, aria, RTL, color contrast |

Most bugs come from missing one axis. Use the QC checklist to verify all 7
before shipping.

## Two-tier Figma model

| Tier | Owns | Examples |
|---|---|---|
| **Master component** | Structure + spacing tokens | `button/default/height`, padding bindings, slot wrappers |
| **Variant component** | Color + state tokens, opacity | Per-state fills/strokes, selected/focus overrides, T3-mode lock |

A variant is a thin wrapper containing one master instance. Color overrides are
applied to the **instance** (not the wrapper), so the wrapper has no fill of
its own. This keeps structural and visual concerns separated.

## Variable hierarchy

```
T0 Primitive Colors          (--blue-500 = #3366f0)
  ↓ alias
T1 Color Tokens              (--brand-500 = T0.--blue-500)
  ↓ context
T2 Surface Context           (default/component/bg-default = T1.--neutral-100)
T3 Status Context            (component/bg-default = T1.--brand-500)  // mode-locked
  ↓ component
comp-size                    (button/default/height = 44)
component CSS tokens         (--btn-bg = var(--{t2|t3}-component-bg-default))
  ↓ internal
component CSS internal vars  (--_btn-bg = var(--btn-{variant}-bg))
```

Every level above only **aliases** the level below — never hardcodes a value.
Project themes override at T1; surface modes switch at T2/T3; component
overrides happen at the public token layer.
